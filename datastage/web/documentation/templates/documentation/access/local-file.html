Setting up local file access

Windows

Data files are accessed from Windows as a network share:

\\{{ hostname }}\data
A username and password will be required to complete a connection. The username entered should be in workgroup {{ settings.GROUP_NAME }}; e.g. {{ settings.GROUP_NAME }}\username.

There are many ways to connect a share in Windows. One is to use Windows Explorer's "Tools > Map Network Drive..." menu command.

On Windows 7, click on the "Start" button, then click on "My Computer", and one of the toolbar options displayed is "Map network drive".

Once connected, a user will be able to read/write files in their own private area, and also to read all files in the shared and collab areas.

Mac OS

Data files may be accessed from your Mac as a Windows network share:

smb://{{ hostname }}/data
A username and password will be required to complete a connection.

To connect a Windows share from your Mac, use the Finder "Go > Connect to server..." menu command (or COMMAND-K shortcut), and enter

smb://username@{{ hostname }}/data as the server address.
The system will then prompt for the specified user's password.

Once connected, a user will be able to read/write files in their own private area, and also to read all files in the shared and collab areas.

Linux

Data files can be accessed from Linux as a network share:

If using GNOME, use the "Places > Connect to server ..." menu command. A popup window will appear. From the "Service type" drop-down menu, select "Windows share". Under "Server" enter {{ hostname }} and under "Share" enter data. Under "Username" enter username and under "Domain" enter {{ settings.GROUP_NAME }} and then click on "Connect". Another popup window will then appear, into which the password for username should be entered.

Once connected, a user will be able to read/write files in their own private area, and also to read all files in the shared and collab areas.

For advanced users, it is also possible to mount the filesystem directly, using the command:

mount.cifs //{{ hostname }}/data mountpoint -o rw,user=username,password=userpass,nounix,forcedirectio
where mountpoint has already been created.

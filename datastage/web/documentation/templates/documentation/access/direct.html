Direct file access from DataStage Linux host

It is possible for research group members to log in directly to the DataStage Linux host and access the file system directly. Generally, this is not necessary and is of use only to those familiar with Linux commands. SCP can sometimes be used to read and write files in circumstances where other methods cannot be used, and has the advantage of being a very secure mechanism for remote file access. There are a small number of administrative tasks that depend on direct system access, though we plan to provide simple web interfaces for these tasks in due course.

Protocols used for direct access are:

  • SSH (Secure Shell: creates a terminal command session), and
  • SCP (Secure Copy: copies files).

Once logged in, a user will be able to read/write files in their own private area, and also to read all files in the shared and collab areas.

Direct access tools for Windows

For Windows there are several freely available tools, including:

  • PuTTY to connect using a SSH login session (http://www.chiark.greenend.org.uk/~sgtatham/putty/),
  • WinSCP to perform SCP file transfers to/from the Windows machine (http://winscp.net/),

Direct access tools for MacOS

MacOS has a built-in ssh command for creating SSH sessions, which can be used thus from the Terminal application:

ssh username@{{ hostname }}

Finder can be used to find the Terminal application in directory Applications/Utilities.

It also provides a command-line SCP file copying command, which is used similarly:

scp username@{{ hostname }}:/home/username/filename localfilename
scp localfilename username@{{ hostname }}:/home/username/filename

For SCP file copying, there is a shareware GUI application called Cyberduck (http://cyberduck.ch/).

Direct access tools for Linux

Linux has built-in commands ssh and scp which can be run from a terminal window.

Here are some examples of their use:

ssh username@{{ hostname }}
scp username@{{ hostname }}:/home/username/filename localfilename
scp localfilename username@{{ hostname }}:/home/username/filename

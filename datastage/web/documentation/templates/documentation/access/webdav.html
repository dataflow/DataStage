Accessing data using WebDAV

Selected DATASTAGE data files may be accessed using the web file access protocol, using one of the following forms of URL:

  • http{{ is_secure|yesno:"s," }}://{{ hostname }}{% url browse:index "private/" %}
  • http{{ is_secure|yesno:"s," }}://{{ hostname }}{% url browse:index "shared/" %}
  • http{{ is_secure|yesno:"s," }}://{{ hostname }}{% url browse:index "collab/" %}

A username and password will be required. Access controls are enforced as described for local network file access page.

Web file access (WebDAV) support is slower and sometimes less reliable than local network file sharing. It is provided for two main reasons:

  1. It can be used from a system outside the Zoology Department network.
  2. It can be used by web browser based applications used for annotation, visualization and other tasks.

Web file access using Windows

Currently, the Windows web file access software does not work with the DATASTAGE server and file protection arrangements — we'll be looking in to this.

Web file access using MacOS

To connect a web file system share from MacOS, use the Finder Go → Connect to server… menu command (or ⌘-K shortcut), and enter one of the URLs shown above as the server address.

The system will then prompt for a username and password.

Web file access using Linux

Under Linux with GNOME, the DATASTAGE web file system content can be accessed in similar fashion to a local file system, but using the web address.

Use the Places → Connect to server… menu command. In the pop-up window, from the Service type drop-down menu, select {% if is_secure %}Secure WebDAV (HTTPS){% else %}WebDAV (HTTP){% endif %}. Under Server enter {{ hostname }} and under Folder enter data/. Fill in the username and password fields with your local credentials for this site.

An alternative mechanism for reading and writing files under Linux is to use the cadaver command-line utility, specifying one of the URL forms listed above.
